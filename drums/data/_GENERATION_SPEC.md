# ドラマー知識検定 カテゴリ生成仕様書（サブエージェント共通）

この仕様に**厳密に**従い、担当カテゴリの2ファイルを生成する。
既存の `data/cat01_basic.js`（本文）と `data/cat01_references.js`（参照表）を**お手本**として必ず読むこと。

## 生成物（担当カテゴリ cat_NN につき2ファイル）

### 1. `data/catNN_references.js`
```js
// cat_NN(カテゴリ名)の参照(図解)テーブル。
window.REFERENCES_CATNN = [
  { id: "ref_xxx_01", type: "table", title: "…", columns: ["…","…"], rows: [["…","…"], ...], linkedQuestionIds: ["q_XXXX", ...] },
  ... (8〜10個)
];
```
- `REFERENCES_CATNN` の NN は2桁大文字（cat_05 → CAT05）。
- 各テーブルは「名称・特徴・用途」等の一覧。関連する複数の問題から `referenceId` で参照される。
- `linkedQuestionIds` には、その表を `referenceId` に指定した問題のIDを列挙。

### 2. `data/catNN_<slug>.js`（本文）
```js
// cat_NN(カテゴリ名) 100問 / 24小枠。難易度配分: 初級30/中級25/上級25/プロ20。
window.SUBSETS_CATNN = [ …24個… ];
window.QUESTIONS_CATNN = [ …100問… ];
```

## 小枠(subset)の構造（合計24小枠・100問）
- 初級: `cat_NN_beginner_01` 〜 `_10`（各3問）＝30問
- 中級: `cat_NN_intermediate_01` 〜 `_05`（各5問）＝25問
- 上級: `cat_NN_advanced_01` 〜 `_05`（各5問）＝25問
- プロ: `cat_NN_pro_01` 〜 `_04`（各5問）＝20問

subsetオブジェクト:
```js
{ id: "cat_NN_beginner_01", categoryId: "cat_NN", title: "…その1", difficulty: "初級", questionIds: ["q_XXXX","q_XXXX","q_XXXX"], bestScoreRaw: 0, bestScoreDisplay: 0, perfectCount: 0, tier: null }
```
difficulty は必ず `"初級"/"中級"/"上級"/"プロ"` のいずれか。

## 問題(question)オブジェクト
```js
{ id: "q_XXXX", subsetId: "cat_NN_…", type: "choice4", difficulty: "初級", text: "…?", media: null, choices: ["正解","誤答","誤答","誤答"], correctIndex: 0, explanation: "…", referenceId: "ref_…_01", points: null }
```
- **`correctIndex` は必ず 0**（正解を choices[0] に置く。表示時にアプリ側が自動シャッフルする）。
- `type`: `choice4`(4択) / `choice3`(3択) / `trueFalse`(○×) / `situational`(状況判断・4択) / `notation`(譜面・4択) / `illustration`(図・4択)。
- `referenceId`: 関連する参照表があれば指定、なければ `null`。おおむね半数の問題に付ける。
- `points`: 常に `null`。
- `text` 末尾は問い（「〜はどれか?」等）。

## 画像付き問題（各カテゴリ最大5問まで）
type を `illustration` か `notation` にし、media を:
```js
media: { kind: "illustration", assetStatus: "placeholder", placeholderId: "img_ph_catNN_xxx", altText: "…の図(仮素材)" }
```
（素材は未調達のためプレースホルダー。5問以内に抑える。）

## 【最重要】誤答設計パレット
各選択式問題は「①正解＋②〜⑨から選んだ3つの誤答」で構成する。**1問ごとに違うタイプの組み合わせ**を選び、単調さを避ける。
- ① 正解
- ② 正解とは逆の意味（特性・手順を反転）
- ③ 別の実在する技術・知識・機材（この問いには当てはまらないが、それ自体は正しい）
- ④ 架空の技術・知識・機材（もっともらしいが実在しない機能・名称）
- ⑤ そもそも論（前提をずらす・논点を回避する、一見もっともらしい主張）
- ⑥ 技術や理論のすり替え（別概念の定義を、この用語の説明にすり替える）
- ⑦ 機材の加工による対応（削る・穴を開ける等、標準的でない加工を答えにする）
- ⑧ 誤った手順（正式な手順であるかのように書いた、実際は誤りの段取り）
- ⑨ 専門用語の羅列（用語を並べて有能そうに見えるが、内容がズレている）

### 厳禁（捨て選択肢）
次のような「一読して明らかに×」の選択肢は**絶対に使わない**:
- 「〜は無関係」「気のせい」「見た目だけ」
- 「練習が不要になる」「暗記するほど自動的に上達」「会話のネタ」
- 「音が自動的に良くなる」「機材が自動的に増える」
- 「所有数」「暗記の速さ」「記憶の速さ」等を問いの答えにする

### 「違いを問う」問題の鉄則
「AとBの違いは?」型は、**4択すべてがAとBの両方を説明する同じ構造・同じ長さ**にする（正解以外は定義を入れ替える等）。

### ○×(trueFalse)問題
choices は `["誤り","正しい"]` または `["正しい","誤り"]`。**「誤り」が正解の問題を全体の3〜5割**入れる（もっともらしいが誤った記述を作る）。correctIndex は必ず 0。

## 難易度の作り分け
- 初級: 名称・基本の事実（choice3/choice4/trueFalse中心）
- 中級: 仕組み・比較・「違いを問う」型
- 上級: 応用・状況判断（situational を混ぜる）
- プロ: 現場判断・「なぜその選択か」。メタな理由問題も**必ずパレットで**作る（正解の周辺にある、もっともらしいが的を外した理由を誤答にする）

## 【追加要件】歴史的背景・ジャンルのルーツを織り込む
各カテゴリに、担当トピックに関わる「楽器・ブランド・奏法の歴史的背景」や「音楽ジャンルのルーツ」を扱う問題を**5〜8問**含めること（中級〜プロに配分）。単なる年号暗記でなく、「なぜそうなったか/どう影響したか」を問い、誤答はパレット②〜⑨で作る。
カテゴリ別の観点例:
- cat_05 セッティング: 配置・人間工学の考え方がどう定着してきたか
- cat_06 メンテナンス: ヘッド素材の変遷(カーフ/皮→プラスチック)、チューニング文化の背景
- cat_07 音作り: ジャンル固有サウンドの成り立ち(ジャズのブラシ、モータウン、ロックの大音量化、ゲート/デッド化の流行等)
- cat_08 演奏技術: ルーディメントの軍楽・マーチング起源、各奏法の系譜と広めた人物
- cat_09 演奏表現・アレンジ: ジャンル慣習のルーツ(ニューオーリンズのセカンドライン、スウィング、ボサノヴァ、ファンク、ディスコの4つ打ち、ヒップホップの打ち込み影響等)
- cat_10 コンビネーション: 時代・ジャンルごとに機材と奏法の組み合わせがどう最適化されたか
- cat_11 トラブル: 昔の機材事情と現代の解決策の対比
- cat_12 現場判断: 録音技術の変遷が現場判断に与えた影響
- cat_13 思考: ジャンルの歴史的文脈が「なぜその一打か」に与える影響
史実は正確に。曖昧な場合は断定を避け、通説として扱う。

## 解説(explanation)
1〜2文。可能なら誤答が何タイプ(②〜⑨)でなぜ違うかに触れる。

## 【重要・既存問題を修正する場合の追加ルール】
既存の問題(text/choices/explanation)を差し替える、または新規パターン(あべこべ型・架空捏造型等)を既存の問題枠に上書きする場合、必ず次を守ること:

1. **referenceIdの内容一致を確認する**: 元のreferenceIdが指す参照表(catNN_references.js)の実際のtitle/rows/columnsを読み、新しい問題内容と話題が一致しているか確認する。一致しない場合は次のいずれかを行う:
   - そのカテゴリ内に話題が一致する既存の参照表があれば、そちらのIDへ差し替える
   - 一致する表が無ければ `referenceId: null` にする(無理に既存の表へこじつけない)
2. **linkedQuestionIdsを両方向で同期する**: referenceIdを変更する場合、
   - 変更前の参照表の`linkedQuestionIds`から該当の問題IDを削除する
   - 変更後の参照表の`linkedQuestionIds`に該当の問題IDを追加する
   (追加のみ・削除のみの片手落ちは不整合を生むため厳禁)
3. **カテゴリのテーマ範囲を逸脱しない**: 例えば cat_02(楽器本体編)は「ドラム本体・シェル・ハードウェアの物理特性」を扱うカテゴリであり、奏者の身体動作(ストローク・フットワーク等)は範囲外。新しい問題を作る際は、必ず該当カテゴリの`_references.js`のtitle一覧を確認してからテーマを決めること。
